================================================================================
STEP 9: TROUBLESHOOTING COMMON ISSUES
================================================================================

ISSUE: Webhook returns 400 or 500 error
-----------------------------------------
CAUSE: Wrong webhook signing secret
FIX: 
  1. Get the exact secret from Stripe > Webhooks > DataGuardianPro-Test-Webhook
  2. Update STRIPE_WEBHOOK_SECRET in your environment
  3. Restart webhook server

ISSUE: Webhook not receiving events
------------------------------------
CAUSE: URL not accessible or firewall blocking
FIX:
  1. Test health endpoint: curl https://dataguardianpro.nl/webhook/health
  2. Ensure port 5001 is open on firewall
  3. Check nginx/reverse proxy configuration

ISSUE: Payment succeeds but tier not updated
---------------------------------------------
CAUSE: Webhook processed but database update failed
FIX:
  1. Check server logs for database errors
  2. Verify DATABASE_URL is correct
  3. Check user exists in database

ISSUE: 37% error rate on webhook
---------------------------------
This is for the LIVE webhook (DataguardianproWebhook), not the test one.
The test webhook shows 0% error rate - this is correct.
Live webhook errors are because you're using test mode keys with live webhook.

STEP 10: SWITCH TO LIVE MODE (WHEN READY)
================================================================================

ONLY do this when testing is complete!

1. Switch Stripe Dashboard to Live mode (toggle off Sandbox)
2. Get your LIVE API keys:
   - Go to Developers > API keys
   - Copy the Secret key (starts with sk_live_)

3. Get LIVE webhook signing secret:
   - Go to Developers > Webhooks
   - Click on "DataguardianproWebhook" (the live one)
   - Copy the Signing secret

4. Update environment variables:
   STRIPE_SECRET_KEY=sk_live_your_live_key
   STRIPE_WEBHOOK_SECRET=whsec_live_webhook_secret

5. Restart your servers
