1. GETTING STARTED
================================================================================

After logging in, go to "New Scan" from the sidebar navigation.

COMMON SETTINGS FOR ALL SCANNERS:
- Select Region: Netherlands, Germany, France, Belgium, EU, or Global
  (This determines which region-specific GDPR rules are applied)
- Free trial users get 3 scans. Upgrade for unlimited scans.
- Paid subscribers can download HTML reports after each scan.

2. CODE SCANNER
================================================================================

PURPOSE: Scan source code repositories for PII, secrets, and security
vulnerabilities with GDPR compliance checking.

HOW TO USE:
1. Select "Code Scanner" from the scan type dropdown
2. Choose your Source Type:
   a) Upload Files - Upload code files directly
   b) Repository URL - Enter a Git repo URL (recommended)
   c) Directory Path - Enter a local directory path

INPUT OPTIONS BY SOURCE TYPE:

  Upload Files:
  - Click "Upload Code Files" and select files
  - Supported: .py, .js, .ts, .java, .cs, .php, .rb, .go, .txt,
    .json, .yaml, .yml, .xml, .html, .css
  - Multiple files allowed

  Repository URL:
  - Git Repository URL: paste your repo URL
    Example: https://github.com/user/repo
  - Branch: which branch to scan (default: main)
  - Access Token: optional, needed for private repositories

  Directory Path:
  - Enter the full path to the code folder on your system

SCAN OPTIONS (checkboxes):
  [x] Fast scan (recommended for large repos)
      - 3-5x faster, scans up to 100 priority files
      - Uncheck for thorough scan (up to 500 files)
  [x] Include comments - scan code comments for PII
  [x] Detect secrets - find API keys, passwords, tokens
  [x] GDPR compliance check - check for GDPR violations
  [x] Generate remediation - provide fix suggestions

CLICK: "Start Code Scan" button to begin

RESULTS SHOWN:
- Files Scanned count
- Lines Analyzed count
- Findings count with severity (Critical/High/Medium/Low)
- Download HTML Report button (paid users)

5. DATABASE SCANNER
================================================================================

PURPOSE: Connect to databases and scan for personal data across
tables and columns.

HOW TO USE:
1. Select "Database Scanner" from the scan type dropdown
2. Choose your Database Type
3. Enter connection details
4. Test connection first, then scan

DATABASE TYPES SUPPORTED:
- PostgreSQL
- MySQL
- SQL Server
- Oracle
- MongoDB

CONNECTION FIELDS:
  Left column:
  - Host: database server address (e.g., localhost)
  - Port: auto-filled based on database type
    (PostgreSQL=5432, MySQL=3306, SQL Server=1433,
     Oracle=1521, MongoDB=27017)
  - Database Name: name of the database to scan

  Right column:
  - Username: database login username
  - Password: database login password
  - Use SSL: checkbox (enabled by default for security)

BUTTONS:
  "Test Connection" - verify connection works before scanning
  "Start Database Scan" - begin the full scan

RESULTS SHOWN:
- Tables Scanned count
- Columns with PII count
- Findings with severity levels
- Download HTML Report button (paid users)

7. AI MODEL SCANNER (EU AI Act 2025)
================================================================================

PURPOSE: Comprehensive EU AI Act 2025 compliance analysis covering
all 113 articles. Includes risk classification and penalty assessment.

HOW TO USE:
1. Select "AI Model Scanner" from the scan type dropdown
2. The 4-step AI Act Calculator wizard will appear

STEP 1 - SYSTEM INFORMATION:
- System Name: name of your AI system
- Purpose: describe what the AI system does
- Use Case (dropdown):
  Healthcare, Financial Services, Law Enforcement,
  Education, Employment, Social Services, Transportation,
  Customer Service, Marketing, Manufacturing, Other
- Deployment Context (dropdown):
  Production, Development, Testing, Research
- Decision Impact (dropdown):
  High (affects fundamental rights), Medium (business decisions),
  Low (recommendations only)
- Automation Level (dropdown):
  Fully Autonomous, Semi-Autonomous, Human-in-the-Loop, Advisory Only

STEP 2 - DATA & OVERSIGHT:
- Data Processing Scope (dropdown):
  Large-scale, Medium-scale, Small-scale, Individual
- Human Oversight: checkbox for human review capability
- Regulatory Context (dropdown):
  EU-wide, National (Netherlands), Cross-border, Non-EU

STEP 3 - COMPLIANCE REQUIREMENTS:
- Checklist of requirements with checkboxes for each
- Mark which requirements you have already implemented

STEP 4 - RISK ASSESSMENT & RESULTS:
- Company Annual Revenue: enter for penalty calculation
- Company Size (dropdown): options available
- Generates risk classification and compliance report
- Shows penalty risk (up to 35M EUR or 7% of global turnover)

RESULTS SHOWN:
- Risk Classification: Prohibited, High-Risk, Limited, or Minimal
- Compliance score
- Required actions
- Penalty risk assessment
- Phased timeline (Feb 2025, Aug 2025, Aug 2026, Aug 2027)

8. DPIA SCANNER (Data Protection Impact Assessment)
================================================================================

PURPOSE: Guided 5-step wizard for GDPR Article 35 Data Protection
Impact Assessment with Netherlands-specific UAVG compliance.

HOW TO USE:
1. Select "DPIA Scanner" from the scan type dropdown
2. Follow the 5-step wizard

STEP 1 - PROJECT INFORMATION:
- Project Name: name of your data processing project
- Data Controller: organization responsible for data processing
- Processing Purpose: describe why you process personal data

STEP 2 - DATA TYPES BEING PROCESSED (checkboxes):
  [ ] Personal data (names, addresses, etc.)
  [ ] Sensitive/special category data (race, religion, etc.)
  [ ] Biometric data
  [ ] Health data
  [ ] BSN data (Netherlands-specific - Citizen Service Number)
  [ ] Government data

STEP 3 - RISK FACTORS ASSESSMENT (checkboxes):
  [ ] Large-scale processing (1000+ individuals)
  [ ] Automated decision-making
  [ ] Vulnerable data subjects (children, elderly, patients)
  [ ] New or innovative technology (AI, ML, biometrics, IoT)
  [ ] Systematic monitoring (CCTV, location tracking)
  [ ] Cross-border data transfers

STEP 4 - SECURITY SAFEGUARDS (checkboxes):
  [ ] Data encryption (at rest and in transit)
  [ ] Access controls and authentication
  [ ] Data minimization practices
  [ ] Data retention policy
  [ ] Consent mechanisms
  [ ] Data breach procedures

STEP 5 - REVIEW AND GENERATE REPORT:
- Shows risk level (High/Medium/Low) with color coding
- Assessment summary with project details
- Risk score out of 10
- Risk factors identified
- Click "Generate DPIA Report" to create the full report

RESULTS SHOWN:
- Risk Level with color coding
- Risk Score out of 10
- Identified risk factors list
- Downloadable DPIA report

12. DATA SOVEREIGNTY SCANNER (Enterprise/Government Only)
================================================================================

PURPOSE: Cross-border data transfer analysis and sovereignty risk
assessment for GDPR Chapter V compliance.

AVAILABILITY: Enterprise and Government license tiers only.
Other tiers will see an upgrade prompt.

HOW TO USE:
1. Select "Data Sovereignty Scanner" from the scan type dropdown
2. Choose your analysis mode

ANALYSIS MODES (radio buttons):

  a) Upload Configuration
     - Configuration Type (dropdown):
       Terraform (.tf), AWS CloudFormation, Azure ARM/Bicep,
       GCP Deployment Manager, Kubernetes YAML, Generic JSON/YAML
     - Upload your infrastructure configuration file
     - Supported: .tf, .json, .yaml, .yml, .bicep, .template

  b) GitHub Repository
     - Repository URL: enter full GitHub URL
       Example: https://github.com/organization/repo
     - Branch: which branch to scan (default: main)
     - Scan Depth (dropdown):
       Full Repository, Root Only, Specific Directory
     - If "Specific Directory": enter the directory path
       Example: infrastructure/
     - Scans for infrastructure files (.tf, .json, .yaml, etc.)

  c) Manual Input
     - Data Locations:
       * Number of locations (1-10)
       * For each: Region, Provider (AWS/Azure/GCP/On-Premises/Other),
         Service type (Storage/Database/Compute/Analytics/AI-ML)
     - Data Flows:
       * Number of data flows (0-10)
       * For each: Source and Destination

CLICK: "Analyze Sovereignty" button to begin

RESULTS SHOWN:
- Sovereignty Risk level (Green/Amber/Red)
- Cross-Border Transfers count
- Non-EU Access count
- Third Country Processors count
- Sovereignty findings with severity
- Recommendations list
- Download HTML Report button
- Checks: US CLOUD Act risk, data location mapping,
  jurisdictional analysis, GDPR Chapter V compliance

GENERAL TIPS
================================================================================

1. REGION SELECTION: Always set the correct region. "Netherlands"
   applies UAVG-specific rules including BSN validation.

2. FREE TRIAL: You get 3 free scans. After that, upgrade to a paid
   plan for unlimited scanning.

3. REPORTS: HTML report downloads are only available for paid
   subscribers. The on-screen results are available to all users.

4. SEVERITY LEVELS:
   - Critical (Red)  = Immediate action required
   - High (Orange)   = Action needed soon
   - Medium (Yellow) = Should be addressed
   - Low (Green)     = Minor issue, good to fix

5. SCAN HISTORY: All completed scans are saved and can be viewed
   from the "History" page in the sidebar.

6. DASHBOARD: The "Dashboard" page shows aggregated metrics across
   all your scans.

7. RESULTS: After any scan completes, go to the "Results" page for
   detailed findings from your most recent scan.
